Design Considerations

Basic usage for Comos Web is (See NamedPipeTests)
  1. Start COMOS.IO.Exe process. Usage : COMOS.IO.exe [CONSOLE]. Console argument will show a console that can be used for debugging.
  2. Setup a channel for communication
  3. Create a communication object of type Comos.IO.IPC.IServiceContract
  4. Open a Comos database
  5. Do some tasks.
  6. Close COMOS database => Will also exit the COMOS.IO.exe process

COMOS.IO.exe does not take a COMOS connection string as argument. Main reason is that it is very hard if not impossible to know when the process has been initialized and to start using it for tasks. This is because opening a COMOS session takes a very long time (many seconds) and NamedPipe technology has no way to detect when the host is ready.
